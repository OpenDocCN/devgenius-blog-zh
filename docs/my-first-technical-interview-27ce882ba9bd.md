给我的任务是使用我选择的 JavaScript 或 Ruby 编写一个方法或函数，它将接受一个字符串输入，输出相同的字符串，但都被打乱了。有人告诉我不要担心输入数据类型，因为我们可以假设总是会提供一个字符串，也不要担心任何特殊字符或空格。我必须记住的唯一规则是，我不能使用 shuffle 方法，这种方法只在 Ruby 语言中可用。Shuffle 是一个 array 类方法，它接受一个数组并随机移动元素。

他们指导我打开我最喜欢的文本编辑器(我喜欢 VS 代码)，并在我准备好的时候开始。由于我无法测试我的代码，这基本上是相当于白板的缩放隔离。

令我非常沮丧和恐惧的是，这个解决方案没有成功，原因有几个。我犯了很多新手的错误，混淆了数据类型，一时冲动没有考虑返回值。

话虽如此，我想深入我代码中的错误，并展示我重构的解决方案(当然是在面试之后)。

第一个——我的错误最多的 for 循环。我在条件语句中声明了一个变量`i`，但从未在循环块中使用过它。这不一定是错的，因为它不会导致错误，但它也是不对的。接下来，在我的第一个`splice()`参数中，我没有考虑到数组的长度会随着每次循环而变小，所以我最终得到了一些空的数组值。我应该做的是在`splice()`论证中使用`arr.length`而不是`str.length`。最重要的是，我应该意识到`splice()`为每个字符创建了一个新的数组，所以当我将`char`推入`new_arr`时，我添加的是包含字符串值的数组，而不是普通的字符串。

第二，紧接着循环之后，我把`new_arr.join(‘’)`。这实际上什么都不做，因为`join()`是一个非破坏性的方法，这意味着它不会改变原始数组。如果我把它赋给一个变量，我就可以在它下面的条件语句中使用它，尽管它仍然是无意义的。

第三，在我的 if 语句条件中，我比较了`new_arr`(一种数组数据类型)和`str`(一种字符串数据类型)。这将永远不会相等，因此`new_arr`将总是被返回，而不是我想要的字符串。

有一件事我很高兴，那就是我知道我需要使用递归来确保我不会返回原来的字符串。我对此感到自豪，因为我通过写关于递归的博客了解了递归，并且能够用最少的经验实现它。写一些我很难理解的概念帮助我了解了更多，我实际上在一次采访中看到了真正的证据。
