# 你的面试准备很糟糕。这不是你的错。

在准备编码面试时(尤其是在大型科技公司)，你会听到的黄金建议是学习数据结构和算法。但是哪个 DSA 呢？又到了什么程度？这些问题都没有答案。如果我们看看上面的脸书面试准备指南，它只是说“充分了解”。但这到底意味着什么呢？人们一生都在研究图形、递归等。DFS 出现在图遍历、语法检查、排序等等。能指望任何一个正常人知道所有这些，在一个高水平上，足以保证一个“完全的理解”吗？看看 Leetcode 上可能出现的问题的数量，很明显没有明确的答案。

![](img/560dc0005e8121eb05411f978946d084.png)

如果你想要一份被证明能帮助你赢得面试的时事通讯，看看[简单编码面试](https://codinginterviewsmadesimple.substack.com/)。评论和经过验证的跟踪记录说明了一切。

编码面试的准备是一门技能。一项依赖于许多支柱的技能，但仍然是一项独特的技能。几乎没有人这样对待它。很有可能，你也不是。这导致你在面试准备中犯错误，耗费你大量的时间和精力。如果你已经花了几周的时间学习像动态编程这样的数据结构/算法/范例，但是两天后就忘记了一切，你就犯了这个错误。让我们开始吧。

# 如何发展技能

类比+比较对于解释想法超级有用。因此，让我们将编写面试代码的技能与我们都必须学习的另一项技能进行比较，**学习语言**。学习语言的阶段是-

1.  **学习字母/发音**——在此之前什么都不能进行。对于编码面试来说，这堪比学习 DSA/基础理论。
2.  **理解基本单词/句子/结构**——这是我们开始把语言作为一种语言来使用的时候。在这个阶段，你开始理解上下文。对于编码面试来说，这是人们从网上复制解决方案/代码来学习它们如何工作的阶段。
3.  用简单的方式表达自己——这是语言变得有用的时候。在您的编码面试中，这是您能够解决较简单问题的阶段。
4.  精通——你可以把复杂抽象的想法用有用的方式表达出来。对于语言来说，那就是交流。对于软件面试，it 部门会拿出解决方案的代码。

目前没有争议。很明显，跳过这些步骤会给你的生活带来不必要的困难。人们基本上同意。大多数人建议先跳 DSA。然而，他们做这件事的方式是错误的。

# 不良 DSA 学习实践

当谈到学习 DSA 时，大多数人遵循以下步骤——1)找到一个用他们的编程语言介绍 DSA 的备忘单/视频(以及时间复杂性之类的东西), 2)尝试使用 DSA 解决简单的 Leetcode 问题。听起来像你吗？那么，问题出在哪里呢？

![](img/43e6cffbc721292631111e4b5686b42b.png)

在调查中得到了这个答案。对这个推荐了一个惊人话题的人大声喊出来。请务必填写我在本文末尾留下的调查。

将此与我们在上一节中讨论的技能发展之旅进行比较。小抄帮你度过了第一阶段。简单的问题在第三阶段。第二阶段呢？你基本上跳过了。现在你能明白为什么这有这么大的问题了吗？你已经知道 DSA 是什么了。接下来，你直接跳到尝试实现它。你真的伤害了你自己的准备。想象一下，在学完字母表(第一阶段)后，甚至不理解简单的时态和动词(第二阶段)的情况下，试图立即向我描述你的一天(第三阶段)。

替代方案是什么？记住，第二阶段是关于上下文的。它是关于理解事物之间的联系。对于 DSA 来说，这需要理解为什么特定的算法/数据结构存在，以及它解决什么问题。请记住，所有这些算法和数据结构都是为了处理特定问题而创建的。了解了历史，你就能掌握数据结构。
