帮助您开始编程的优化方法

原文：https://blog.devgenius.io/an-optimized-approach-to-get-you-started-with-programming-5a77ded29e97?source=collection_archive---------16-----------------------#2022-05-29

你有开发人员的朋友，或者你只是在看到开发人员的生活方式后，想开始工作。它对你说话，你渴望开始。你告诉自己一个月的时间足够精通却不知道从何下手。

需要多长时间？

一个多月；我可以向你保证。这是每个人都要经历的。不管是什么原因，大量的前程序员认为一个月就可以成为开发者；有的比较保守，说三。当经验丰富的开发人员想要学习一门新语言时，三个月通常是一个乐观的时间框架。你将面临近一到两年的刑期。你每天投入的时间越多，你就离一年的目标越近。

从哪里开始？

如果你在学校学习编程，那就是你开始的地方。专注于你的项目，努力成为一个超水平的人。当你得到一个项目时，假装你得到了报酬。全面完成项目，做任何额外的事情。我记得在我参加移动应用程序开发课程时，有一个这样的场景。尽管我已经有多年的编程经验，但我只有很少的移动开发经验。对于最终项目，我们需要挑选 3 个特性，并把它们实现到我们选择的应用程序中；有 40 种可能的选择。当每个人选择他们的三个时，我实现了所有的 40 个。我不是在寻求额外的荣誉或鼓励。我上那门课是为了学习移动应用开发，想充分利用它。不断重复这一点，专注于计算机科学的基础知识。当你真正理解编程的基础时，学习一门语言/框架会容易得多。

如果你没有参加任何编程课程，你有一个优势和一个劣势。只要你知道自己的最终目标，以及如何达到目标，你就可以跳过大学要求的所有核心课程，把时间花在编程上。你每天吃和呼吸代码 8 小时，将加速你的旅程。

有几个缺点。你更有可能跳过困难的概念，因为你对完成你的任务没有责任。你将会构建这个计算器一百次，但是你将会不愿意超越它。另一个缺点是你没有课程可以遵循。大学课程旨在相互借鉴。你可能会开始跳到你还没有准备好的概念上，并且大部分时间会觉得自己快要淹死了。你不会理解任何人实际上是如何学习编程的，你的大脑会不断告诉你放弃，因为这不适合你。这时你应该后退一步，意识到你没有采取正确的方法。

方法是什么？以下是我向所有询问我的新开发人员推荐的内容:

  • 获取编程入门书籍。
  • 研究你感兴趣的节目类型。你会选择移动应用程序开发道路、web 开发道路、嵌入式系统道路还是其他道路？
  • 一旦你知道了你的道路，是时候研究一下所使用的编程语言的类型了。
  • 购买一两本关于编程语言入门书籍。同时跟随介绍性的 YouTube 教程。
  • 全面了解 YouTube。是时候找应用教程了。
  • 学习其他必要的编程概念，如版本控制和如何使用项目管理工具。
  • 建立投资组合。构建的应用程序越多越好。
  • 找一份实习或初级工作。从经历中消耗一切。
  • 几年后升到中层职位。再次，消耗一切，并开始观察高层职位的行为。
  • 获得你的高级职位。

你的第一本编程书(1-3 个月)

你会不止一次地阅读这本书。找一本关于 Java 或类似语言的书，严格要求面向对象编程的基础。阅读这样一本书的最佳方式如下:

  • 每天最多读一章。
  • 在你阅读下一章之前，快速浏览你之前的所有章节。
  • 一旦你完成了这本书，从头开始，重读它的全部细节。

你可能会相对容易地掌握前两到三章，尤其是如果你在数学上很用心的话。你将开始想象什么是函数/方法，以及它与数学的关系。

下一章将是残酷的，你会大汗淋漓。刚刚发生了什么？你正在漫游，突然你被巨大的学习曲线击中了。你不知道如何从这里进步。在网上研究特定的话题，并试着理解它们。当你最终完成并且感到紧张时，从头开始。下次你会有更好的理解。也有这样的时候，当你读一本编程书，它说，“就像这样写代码，我们会在后面的章节解释它的意思。”那只是意味着有一些你还没有想到的基本概念。一旦你读到解释它的那一章，然后重读之前的章节，一切都会变得非常有意义。

完成这本书后，休息几天，然后再读一遍全文。这将帮助你巩固概念。你会惊讶于你现在理解了多少，错过了多少，尽管你觉得这本书你已经读了一百遍了。

研究你感兴趣的节目类型(第 4 个月)

你可能想进入移动应用程序开发，但你不知道你不知道什么。研究使用的编程语言的类型。例如，您对构建 Android、iOS 或跨平台应用程序感兴趣吗？这些都需要不同的编程语言。哪里的开发者最开心？你认为未来的自己在哪里？你呼吸苹果产品吗？你想只构建 iOS 原生应用吗？时间决定了你是想学 SwiftUI 还是 UIKit。

是时候研究所使用的编程语言的类型了(4.5-5 月)

想从事网络开发。这是一个全新的难题。在开始之前，你必须了解一些基础知识，比如 HTML 和 CSS。你想做前端还是后端开发者？你知道前端和后端的区别吗？您是在寻找最时尚的编程语言，还是对稳定的企业级语言感兴趣？这些决定需要你通过自己的个人研究来发现。有成千上万的文章和视频，个人用特定的语言描述他们的职业生涯。

购买一两本关于你所选择的编程语言的入门书籍。同时跟随介绍性的 YouTube 教程。(第 6-9 个月)

一旦你知道了自己的道路，以及如何在这条道路上取得成功，是时候买一本关于编程语言的书了。你现在应该知道你的先决条件；如果有的话，你会想先读那些书。例如，如果您想成为使用 ReactJS 的前端开发人员，那么您需要了解 HTML、CSS 和 JavaScript 的基础知识。你可能还想熟悉像 Bootstrap 这样的 CSS 框架或者像 Tailwind 这样的实用框架。只有当你对这些基本原则感到满意时，才是做出反应的时候。

为了帮助你加快学习过程，在你看书的同时，跟着几个 YouTube 教程。这些不应该专注于构建应用程序，而是应该专注于编程语言或框架/库的基础。

全面了解 YouTube。是时候找应用教程了。(第 10-12 个月)

现在您已经理解了这些概念，是时候继续学习并构建一些应用程序了。你以前可以这样做，但是有一个原因可以解释为什么这是个坏主意。如果有人让你重新开发你刚刚花了 6 个小时在 YouTube 上关注的应用程序，你不会这么做。你现在仍然会挣扎，但是你的思想已经足够成型，你可能会有一个大概的想法。尽可能多地浏览这些教程。一旦你完成了它们，试着自己重新创造它们，不需要他们的帮助。如果你被卡住了，谷歌一下你的问题。在堆栈溢出上查找答案。做开发人员做的事情，但不要再跟着做，欺骗自己。

学习其他必要的编程概念，如版本控制和如何使用项目管理工具。(第 13-14 个月)

如果你已经超过了一年，那么你很可能会认真地把编程作为职业。是时候熟悉一下程序员使用的工具了，比如 Git、GitHub、BitBucket、吉拉、Confluence、Asana、Trello 等等。互联网是你的朋友。你研究并练习使用这些工具吗？他们是任何常规工作环境的一部分。这也是开始观看关于典型程序员日常工作的视频的好时机。你会看到像单口站立提到的东西。研究开发人员在站立时做什么。

建立投资组合。构建的应用程序越多越好。(第 15-24 个月)

在观看 YouTube 视频时，您已经构建了一些应用程序。如果你已经把它们制作成你自己的，并且在没有观看那些视频的情况下重新制作了它们，把它们上传到你的 GitHub 账户。查找现实世界的问题，并尝试创建解决这些问题的应用程序。这些不一定是生产就绪的应用程序，但是您应该努力使它们尽可能完美。这个想法是每天练习开发，并使用你在过去一年半中积累的知识。专注于在各种环境中部署应用程序；这将使你的职业生涯更容易，我可以向你保证。

找一份实习或初级工作。从经历中消耗一切。(2-4 岁)

在你开始面试之前，你会想看看模拟面试，并练习回答常见的编程问题。一旦你找到了初级工作，就好好利用它。公司和更高层的员工会知道你没有能力解决困难的任务。他们会一直慢慢逼你。利用这一点，尽可能多地吸收。这是你的第一次工作经历，你会有所挣扎。明白公司不希望你失败。更高级别的开发人员会提供帮助，所以在这个阶段，请务必提出您的问题。这将促进你的学习和事业。

移动到中级位置。再次，消耗一切，并开始观察高层职位的行为。(4-6 岁)

你被提升为中级开发人员；恭喜你。是时候接受更具挑战性的任务，开始规划通往高级开发人员职位的道路了。当你还是初级开发人员时，你没有时间去观察高级开发人员在做什么；你几乎不能专注于你正在做的工作。与你的同事交谈并检查他们的代码。在代码评审中采纳每一个建议，并在你的代码中实现它。你的中层职业生涯会比你想象的更快结束。

获得你的高级职位。(7 岁以上)

时候到了，到时候你会知道的。你将协助初级和其他中级开发人员。任务会变得更加困难，但是你会开始享受挑战。其他时候，任务似乎不费吹灰之力。你将晋升到高级职位。放松点，你成功了。现在怎么办？如果你像大多数开发人员一样，你会寻找过去发展的职业，你会确切地知道下一步该做什么。
