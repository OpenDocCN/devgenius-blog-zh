开始新的开发工作之前要做的 5 件事

原文：https://blog.devgenius.io/5-things-to-do-before-starting-your-new-developer-job-8163ddd664ab?source=collection_archive---------27-----------------------#2020-06-30

恭喜你有了一份闪亮的新工作！你努力学习，通过了艰苦的技术面试，或者完成了一个带回家的项目(“这应该只占你 5 个小时的时间”，但你实际上连续一周每天晚上都在做这件事)，并且成功地谈妥了一份很棒的工作机会和开始日期。

随着你的开始日期悄悄临近，你突然比以往更加紧张:我准备好了吗？

不要惊慌。这里有一些可行的方法让你平静下来。记住，你已经得到这份工作了，准备好大干一场吧！

研究，研究，再研究

你的新团队的技术是什么？Java/JavaScript/TypeScript/Python？上帝保佑，咖啡脚本？你会做前端还是后端代码？AWS 还是 Azure？MongoDB 还是 PostgreSQL？React 还是 Vue.js？

希望你在面试中已经有了好的想法。如果没有，在 LinkedIn / email / Slack 上联系你未来的老板或队友。他们会非常乐意帮助一个热切的未来同事。

温习与你的角色相关的任何东西:从语言基础开始(你还记得如何在现代 JavaScript 中循环对象属性吗？:D )、web 框架、分布式系统、特定数据库、AWS / Azure 概念。如何使用 IDE 进行调试？任何你能拿到的东西。

为新工作做好准备比在第一天上班时匆忙查看 W3schools 了解 flexbox 如何工作要好得多。

使用编码形来提高你的技能

什么是形？https://en . Wikipedia . org/wiki/Kata _(编程)

如果你有一段时间没有写代码了，很可能你已经生疏了。就像任何其他技能一样，编程是一项需要经常练习的技能，这样你才不会肌肉萎缩。做一点编程，最好是用你在新工作中会用到的语言/框架，是非常宝贵的。

再次执行 todo-list 项目，这次是在您将要使用的框架中。解决一些 Leetcode 容易的问题。使用 TypeScript / React 写井字游戏。无论你选择做什么，锻炼你的肌肉，让它们为你的第一天做好准备。

再次读取干净的代码

如果你还没有，读一读《干净的编码者》。它讲述了一个专业软件工程师应该是什么样的。它也给你一些可行的建议，告诉你如何避免常见的错误，做正确的事情。

通常，当我们在一项工作上呆的时间太长时，我们会习惯于现状，忘记如何成为一名优秀的开发人员。见鬼，也许你离职是因为现状太糟糕了。T11】

“我并不真的进行集成测试，因为我们的内部回购并不真的支持集成测试”

“我懒得用棉绒，因为没人用它”

“我复制粘贴了这个类，以遵循现有的代码库结构。我不想重写整个模块。”

一旦我们开始在工作中养成坏习惯，它们就很难改掉，在我们作为一名新的开发人员重新开始之前，需要进行戒毒。

准备设置您的环境

乍一看，这似乎有悖常理。

“如果我连笔记本电脑都没有，我该如何准备设置我的工作环境？”

但是每个开发者都有很多东西。Git 配置。您的 Vim / Emacs 配置和插件。自定义 VSCode 代码片段和快捷方式。您的 IntelliJ 布局。您的终端/我的天啊配置。

为你的工作环境做好准备会让你尽快胜任新的工作，你会给人留下深刻印象。

喘口气

最后，不要忘记你已经得到了这份工作。

放松，深呼吸。

如果你的旧工作很累人(因此有了新的工作)或者你压力太大，花些时间来调整你的精神状态。

读本书，度个假，烤点饼干，试试新的妆容。不管什么能让你开心。

记住，你已经成功了。休息一下，回来后好好表现。
