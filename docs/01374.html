我们以前在“初学者”等级中遇到过类似的代码，但这次挑战要求我们解决它。幸运的是，对我们来说，这并不复杂。

挑战从两个数组a1 and a2开始，每个数组包含五个随机数，还有一个空数组arr。下面是一个 while 循环，只要两个a1 and a2都不为空，它就会执行。如果您分析循环体，它使用pop(从数组末尾)删除数字，并比较这些数字。根据x and y的情况，这些数字被添加到arr。下面是一些帮助我们分析代码的伪代码:

a1.pop  ->  x=4
a2.pop  ->  y=6
push    ->  arr=[6, 4]a1.pop  ->  x=4
a2.pop  ->  y=9
push    ->  arr=[..., 9, 4]a1.pop  ->  x=3
a2.pop  ->  y=4
push    ->  arr=[..., 4, 3]a1.pop  ->  x=0
a2.pop  ->  y=9
push    ->  arr=[..., 9, 0]a1.pop  ->  x=7
a2.pop  ->  y=3
push    ->  arr=[..., 7, 3]arr = [6, 4, 9, 4, 4, 3, 9, 0, 7, 3]

现在我们知道了arr的最终值，挑战要求我们求解R = arr[4]，这是第 4 个索引处arr中的值，在本例中是4。

重要提示:我们已经分析了上面的整个代码，但是请记住，一旦我们在索引 4 处找到了arr的值，我们就应该停止，因为这是这里唯一重要的事情。
