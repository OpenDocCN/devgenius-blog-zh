概观

Jetpack Compose 是 Android 用于构建原生 UI 的现代工具包。它简化并加速了 Android 上的 UI 开发。借助更少的代码、强大的工具和直观的 Kotlin APIs，快速实现您的应用。它使得构建 Android UI 更快更容易。

Android 不是唯一受支持的平台。首先，你可以在 Web 和 桌面 — 多平台 应用中使用它。

Jetpack Compose 的优势(优点)

  • 编写更少的代码会影响开发的所有阶段。
  • Compose 使用声明式 API，这意味着您需要做的就是描述您的 UI。
  • Compose 与您现有的所有代码兼容。
  • 它易于更新，易于测试。
  • 它很容易与 Android 中现有的视图兼容。
  • 它提高了发展速度。
  • 它删除了 findViewById 或 ViewBinding 引用的样板文件。

Jetpack Compose 的缺点(缺点)

  • 这种解决方案的缺点是每次代码改变时都要重新渲染预览，构建速度不是很快。
  • 呈现更改比 xml 慢一点。(尝试在关闭调试日志的情况下构建发布版本，应该可以正常工作😊)
  • 有些组件不受支持，有些功能即将推出。你可以看一看 路线图 。

环境设置

为了能够构建应用程序，你需要 Android Studio 最低版本Bumblebee (2021.1.1)

应用程序结构

干净架构的一个主要好处是在整个应用程序和源代码中有清晰的导航。查看项目的根文件夹，下面的结构变得清晰。

建筑 Src

buildSrc层负责依赖关系管理。它通过 Kotlin 在一个地方控制和管理所有依赖项。

应用

app层负责公共和一般属性。

  • 组件:负责总视图。
  • widget :负责使用 app 的常用视图组件。
  • 主题:定义主题、颜色、字体和资源文件。

供应者

provider层负责导航、资源和主题提供者。

图书馆

libraries层负责基础结构。

  • 框架:包含不同模块和基础结构可以使用的不同工具。
  • jetframework :包含不同的工具，这些工具可以由不同的模块和基础结构使用，仅用于 Jet Compose。
  • 测试:包含测试工具的定义。

数据

data层负责为领域层选择合适的数据源。它包含域层中声明的存储库的实现。

数据层的组件包括:

型号:

  • -dto :定义 ui 模型的 dto，同时进行domain、response和entity模型之间的数据转换。
  • -local :定义 SQLite 数据库的模式。
  • -remote :定义网络响应的 POJO。

本地:

这负责使用空间执行缓存操作。

远程:

这负责执行网络操作，例如使用改进定义 API 端点。

存储库:

负责向域层公开数据。

领域

这是应用程序的核心层。domain层独立于任何其他层，因此】领域业务逻辑可以独立于其他层。这意味着其他层的变化不会影响领域层，例如，屏幕用户界面(表示层)或数据库(数据层)的变化不会导致领域层内的任何代码变化。

域层的组件包括:

  • 用例:它们封装了一个单独的动作，比如从数据库中获取数据或者向服务发送数据。他们使用存储库来解决他们应该做的动作。它们通常覆盖操作符invoke，所以它们可以作为一个函数被调用。

介绍会；展示会

features层包含向用户显示信息所涉及的组件。这一层的主要部分是视图(活动、合成)和视图模型。

技术堆栈

这个项目使用了 android 生态系统中许多流行的库、插件和工具。

-依赖性

撰写

  • 材质 —用现成的材质设计组件构建 Jetpack Compose UIs。
  • 基础 —使用现成的构建块编写 Jetpack 组合应用程序，并扩展基础以构建您自己的设计系统。
  • UI —构成 UI 与设备交互所需的基本组件，包括布局、绘图和输入。
  • ConstraintLayout—constraint layout-Compose 1.0 提供了 Jetpack Compose 中的 constraint layout 功能。
  • 生命周期-视图模型 —执行动作以响应另一个组件(如活动和片段)的生命周期状态的变化。
  • 分页 —分页库让你更容易在应用的 RecyclerView 中逐步优雅地加载数据。
  • Lottie—Lottie 是一个 Android 和 iOS 的移动库，解析 Adobe After Effects 动画用 Bodymovin 导出为 json，并在移动上原生渲染！
  • Coil—kot Lin 协程支持的 Android 镜像加载库。
  • 导航 —用于类型安全的 Jetpack 撰写导航的注释处理库，没有样板文件。

伴奏

  • SwipeRefresh——一个库，提供了一个提供滑动刷新 UX 模式的布局，类似于 Android 的 SwipeRefreshLayout。
  • systemui Controller—系统 UI 控制器提供易于使用的实用程序，用于更新 Jetpack Compose 中的系统 UI 栏颜色。
  • Insets—Insets for Jetpack Compose 吸取了许多推动 Insetter for views 的想法，并将它们应用到 composables 中。
  • 占位符 —一个库，在加载内容时为显示“占位符”UI 提供一个修改器。
  • 导航 —为 Jetpack 导航作曲提供作曲素材支持的库。此功能具有可组合的底部工作表目的地。

喷气背包

  • Android KTX—为 Jetpack 和 Android 平台 API 提供简洁、惯用的 Kotlin。
  • AndroidX—对原有 Android 支持库的重大改进，不再维护。
  • 生命周期 —执行动作以响应另一个组件(如活动和片段)的生命周期状态的变化。
  • ViewModel —旨在以生命周期意识的方式存储和管理 UI 相关数据。ViewModel 类允许数据在配置更改(如屏幕旋转)后仍然存在。
  • 房间 —在 SQLite 上提供一个抽象层，用于离线数据缓存。
  • 分页 3 —分页库让你更容易的逐步加载数据。
  • 匕首刀柄 —依赖注入库。
  • 谷歌-KSP —科特林符号处理 API
  • 改进 —类型安全的 http 客户端，支持开箱即用的协程。
  • ok Http-Logging-Interceptor—记录 HTTP 请求和响应数据。
  • 协程 —对协程的库支持。
  • 流 —流建立在协程之上，可以提供多个值。从概念上讲，流是可以异步计算的数据流。
  • 材质设计 —打造令人敬畏的美丽 ui。
  • 协程 —协程库支持，提供测试中使用的运行阻塞协程生成器。
  • Timber —一个带有小型可扩展 API 的日志程序，它在 Android 的普通日志类之上提供了实用程序。
  • Moshi—一个面向 Kotlin 和 Java 的现代 JSON 库。
  • Chucker—Android 的一个 HTTP inspector&ok HTTP(类似 Charles 但在设备上)。
  • grad le Kotlin DSL—让我们轻松管理所有模块的依赖关系。
  • 闪屏 — Android 12 增加了闪屏 API，当运行在 Android 12 或更高版本的设备上时，可以为所有应用启用新的应用启动动画。

-测试依赖关系

  • JUnit —一个编写可重复测试的简单框架。它是用于单元测试框架的 xUnit 架构的一个实例。
  • 模仿 —提供 DSL 来模仿行为。从零开始构建以适应科特林语言。
  • Android x——Android x 测试库为测试 Android 应用提供了一个广泛的框架。
  • Robolectric—Android 的行业标准单元测试框架。
  • Turbine—kotlinx . coroutinesFlow.的小型测试库
  • mock web server—用于测试 HTTP 客户端的可脚本化 web 服务器。
  • 协程 —提供有效测试协程的测试实用程序。

-代码分析插件

  • Ktlint —一款内置格式化器的防自行车脱落 Kotlin 棉绒。
  • Detekt—kot Lin 编程语言的静态代码分析工具。
  • 一尘不染 —一个代码格式化程序可以做的不仅仅是发现格式化错误。
  • 版本 —便于确定哪些依赖项有更新。
