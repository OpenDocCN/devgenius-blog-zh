# 尝试完全不同的技术的理由

待在舒适区是人类的天性。名字里有“舒适”是有原因的。话虽如此，众所周知，呆在自己的舒适区里很少能学到东西，如果说我们对技术有什么了解的话，那就是它一直在变化。牢记这一点，你可以用各种方法离开你的舒适区。那么如何选择如何离开自己的舒适区，尝试不同的技术呢？

第一步可能是决定你为什么试图走出你的舒适区，目标是什么？如果目标是对新流程和新技术保持开放，您可以选择在低风险的地方一次混合一项新技术。这种方法在比较不同技术时特别有用。这更接近于一种科学的方法，改变一个变量，比较和对比它对结果的影响。这种方法是有用武之地的，我认为这是一种我们都应该时刻牢记的方法。也就是说，我今天想关注的是一种不太科学的方法，它的目标仅仅是为了学习而学习，没有明确的目标。这种方法是你选择使用完全不同的技术，完全走出你的舒适区。

每个人的具体情况会有所不同。潜在地，你是一个 Python 开发者，这可能意味着你尝试捡起并学习一些关于 Rust 的知识。看看强类型系统是什么感觉。或者你可能从未尝试过函数式编程，你可以试一试。这肯定可以扩展到编程语言之外。尝试新的数据存储、编程模式、消息队列、部署系统等。你永远不知道，你可能会找到比你现在用的更好的东西。

这一实践中最好和最有成效的部分不是寻找替代您当前工具的工具。虽然这种情况可能发生，但以我的经验来看，这种情况不太可能发生。更经常发生的是，通过使用其他技术可以引入一种新的/不同的思维方式，并且您可以将这种学习带入日常工作中。它也非常有助于您对工具箱中的许多工具和技术有一个基本的了解。你可能不经常使用它(甚至从来不使用),但是如果你对某个特定的工具毫无经验，你知道它是这项工作的合适工具的可能性就会大大降低。

当你学习这些其他技术时，了解人们使用这些技术的原因和他们不使用的原因是非常有用的。我们经常寻找信息来证实我们当前的信念，而寻找与我们当前观点相矛盾的信息往往更能提供信息。这并不意味着你需要购买相反的信念，但至少有一个熟悉可以帮助你避免陷阱。也许你使用 Java 并且热爱它。出门求“为什么不用 Java？”。也许你认为没有理由使用关系数据库以外的任何东西，找出人们为什么选择 NoSQL。

科技行业是一个不断学习的行业。你可以自主学习，也可以被迫学习。毫无疑问，控制你的学习会有更好的回报。采取选择挑战你的经历和尝试新事物的方法可以获得很多回报。
