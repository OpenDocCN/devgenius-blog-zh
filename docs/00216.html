每个工程师/架构师都需要知道的关于安全性的25件事

原文：https://blog.devgenius.io/25-things-about-security-that-every-cloud-solutions-architect-need-to-know-d8a0dc3f7456?source=collection_archive---------0-----------------------#2020-05-18

这篇文章是一个系列的一部分(参见第二部分关于数据和第三部分关于网络),所有软件工程师和解决方案架构师需要知道在云上产生一个安全的、可伸缩的、弹性的和架构良好的解决方案。

该列表不是详尽的列表。但是，它应该足够好，可以快速上手。

对于这个列表中的每一项，都有一个附带的链接，主要是引导你到谷歌，并为你设置了正确的关键词。

让我们进入列表，如果你觉得有什么不对，或者想在列表中添加一项，请在下面留下评论。

安全/解决方案架构师需要了解:

  1. 对称和非对称加密之间的区别，不同算法(RSA、DES、Triple DES、AES)使用的加密类型，以及何时使用每种类型。(参见谷歌)
  2. 如何管理加密密钥(客户管理、客户提供和云管理)以及为您的密钥选择哪个选项(参见 AWS )
  3. 什么是秘密管理和密钥管理服务(KMS)及其在云安全中的作用？(参见谷歌)，以及了解加密密钥层级和信任根(参见谷歌)
  4. 静态加密和传输中加密的区别。(参见谷歌)
  5. 了解信封加密及其如何用于保护您的安全密钥，以及加密过程中使用的不同类型的密钥(主密钥、DEK、KEK 等)。).(参见 IBM )
  6. 了解不同的安全模型(RBAC、ACL、IAM)安全上下文中策略、角色和组之间的区别。(参见 AWS )
  7. 了解身份联盟如何使用ADFS或其他服务在云和本地之间工作。(参见 Google )，以及身份感知代理如何在普通的旧VPN上提供更安全的连接(参见 Medium )
  8. 了解AAA(AAA)安全性(认证、授权和审计)。(见谷歌)，以及了解CIA三要素(保密性、完整性和可用性)。(参见什么是)
  9. SSL/TLS握手如何工作，握手过程中何时使用非对称密钥和对称密钥？(参见 Cloudflare )以及SNI如何促进在同一个IP地址上使用多个证书(参见 GlobalSign )
  10. 了解什么是单点登录SSO，SAML、OpenID、JWT和OAuth-2.0是如何工作的，以及不同类型的访问令牌(长期令牌、刷新令牌等)之间的区别。).(参见谷歌和 Youtube )
  11. 知道如何使用HTTP基本身份验证、API密钥、JWT令牌等来保护公共API。(参见谷歌)
  12. 了解SIEM解决方案的作用以及DevSecOps的含义。(参见谷歌)
  13. 了解不同类型的安全威胁以及WAF ( Web应用防火墙)的作用，以抵御这些攻击( DDoS &僵尸网络、中间人(MitM) 、 SQL注入、跨站点脚本XSS 、 CSRF攻击、混合HTTP/HTTPS内容、端口扫描、彩虹表
  14. 了解防火墙的作用，它如何处理入站和出站流量，以及它如何帮助在网络中创建隔离区。(见 Azure )，也知道有状态防火墙和无状态防火墙的区别。(参见 CDW )
  15. 数字签名如何工作，以及在技术上“签署”文档意味着什么。(参见文档设计)
  16. 了解零信任安全模型(参见谷歌)、最小特权原则(POLP)(参见谷歌)、以及安全上下文中的职责分离(SoD)(参见谷歌)
  17. 如何使用Bastion主机(跳跃盒)访问网络，为什么它比其他方式更安全。(参见谷歌)
  18. 知道如何使用PenTest(参见 Google )和Kali Linux(参见 Google )、公钥基础设施PKI(参见 Google )等强化Linux发行版以及硬件安全模块HSM来发现解决方案中的安全漏洞吗？(参见谷歌)
  19. 风险、脆弱性和威胁的区别？。(参见谷歌)
  20. 了解入侵检测系统(IDS)和入侵防御系统(IPS)(及其变体HIDS和NIDS)之间的区别。(参见 DNSStuff )
  21. 编码、加密、哈希、盐析哈希和混淆之间的区别。(参见 Google )，以及匿名化和标记化的区别(参见 Google )，以及什么是数据净化(参见 Blanco )
  22. 多因素身份认证的工作原理，尤其是最常见的实施方式(双因素身份认证)。(参见谷歌)
  23. 对通用隐私立法/标准(GDPR、HIPAA、COPPA、PCI DSS)有较高的认识。(见谷歌)，以及如何准备信息安全审计(见 Dashlane )
  24. 了解安全通信协议，如TLS/SSL、IPSec、SFTP和FTPS。(参见谷歌)
  25. Ping和traceroute的工作原理。(参见 Youtube )

我真的相信知识是产生适当的架构良好的解决方案的第一步，然而，仅仅有知识并不能保证你能开发这样的解决方案，你还需要经验、指南、最佳实践、架构框架、参考架构等等。
