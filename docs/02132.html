网络/解决方案架构师需要了解:

  1. 了解网络中的数据、数据包和帧，以及网络内部如何使用IP和MAC地址进行路由。(参见 Youtube )
  2. 了解什么是网络地址转换NAT，它是如何工作的，以及它的不同变体SNAT、DNAT、PAT和端口转发(见 Youtube
  3. 了解什么是子网划分(参见Youtube)RFC 1918如何使用三个IP类10中的一个为您的子网选择合适的IP范围。*, 172.16.*和192.168。*(参见 IETF )以及何时使用。
  4. 了解不同类型的网络(局域网、无线局域网、广域网(见 Quora ))以及局域网和VLAN的区别(见 Youtube )，还知道集线器、路由器和交换机的基础知识以及它们在网络连接中的作用(见 Youtube )。
  5. 了解网络上不同类型的攻击，如DOS/DDOS/EDOS攻击以及如何保护网络免受它们的攻击(参见 CyNet )，还了解什么是经济DOS (EDOS)攻击以及如何防范它们(参见 ZDNet )，还了解入侵检测系统(IDS)和入侵防御系统(IPS)之间的区别(及其变体HIDS和NIDS)(参见 DNSStuff )
  6. 了解DHCP协议如何获取动态新IP(参见 Youtube )，了解静态IP和动态IP的区别(参见 Google )，还了解地址解析协议(ARP)及其用途(参见 Youtube )，还了解带宽、延迟和速度的区别(参见 Youtube )
  7. 了解OSI模型APSTNDP的7层(7-应用层、6-表示层、5-会话层、4-传输层、3-网络层、2-数据链路层、1-物理层)(参见 CloudFlare )，也知道每一层的典型职责，以及在哪一层运行的标准协议(HTTP - >第7层、TLS/SSL- >第6层、SMB- >第5层、UDP/TCP- >第4层等)。)(参见维基百科)
  8. 了解TCP、UDP等不同网络协议的特点(见 Youtube ，以及FTP、s FTP、FTPS(见 GoAnyWhere )、POP3、SMTP(见 ServerSMTP )，以及它们之间的区别和各自适合的应用，也了解什么是单播、组播、广播(见 Youtube )。
  9. 了解什么是软件定义的网络、它如何工作(参见 Ciena )及其在云网络中的作用。
  10. 了解什么是VPC(虚拟私有云)(参见 AWS )，以及它如何在云上使用，以及如何将VPC划分为子网(参见 AWS )，还了解什么是CIDR块，以及如何选择满足您需求的CIDR范围(参见 AWS )。
  11. 了解VPC对等以及我们为什么需要它(参见 AWS )以及如何使用瞬态网关在perm网络上连接云网络或者将不同的VPC连接在一起(参见 AWS )，还了解Bastion主机(跳转框)以及它们如何被用来访问你的VPC内部的私有实例(参见 Youtube )。
  12. 了解内部数据中心如何使用VPN或私有连接连接到云数据中心，如AWS DirectConnect ，谷歌专用互连和微软 ExpressRoute ，以及如何使用私有连接而不通过公共互联网访问内部云服务，如AWS VPC端点/PrivateLink ，谷歌 PrivateAccess 。
  13. 了解DNS如何工作，以及个人笔记本电脑详细解析IP地址需要经过的不同跳数(浏览器DNS缓存->操作系统DNS缓存->ISP解析服务->根服务器-> TLD服务器->权威名称服务器(参见 Youtube )，还知道DNS如何使用端口53上的UDP进行通信(参见 Google )以及最近向TLS上的安全DNS的过渡(参见维基百科)并知道BIND9是什么(参见ISC【13】
  14. 了解DNS缓存的工作原理和TTL设置的作用(参见 Varonis )，也了解常见的DNS记录类型(CNAME、A、AAAA等。)(参见简单域名系统)。
  15. 了解不同的DNS路由选项(故障转移、延迟、地理定位、多值、加权)(参见 AWS )，还了解什么是DNSSEC(参见 NameCheap )，以及DNS服务何时在TCP和UDP之间切换(参见 ServerFault )。
  16. 了解什么是VPN(见 Youtube )以及它如何使用OSI第三层协议(IPSec协议)与服务器通信。还要了解一些关于OpenSwan的知识，以及它是如何从内核2.3开始成为Linux上的标准IPSec实现的(参见 OpenSwan )，以及如何使用UDP封装IPSec数据包以通过NAT(参见 StackOverflow )，还要了解IPSec隧道及其在站点到站点S2S VPN中的作用(参见 Youtube )，以及点对点P2S(参见微软)，还要了解何时使用IPSec vs
  17. 了解互联网协议套件的4层(1:应用，2:传输，3:互联网，4:链接)(见维基百科)互联网如何使用TCP/IP连接客户端和服务器。
  18. 了解所使用的标准互联网协议，例如在网络间广播路由的BGB协议(参见 Youtube )和eBGB与iBGB的区别(参见 Youtube )，以及MPLS如何使用标签来加速路由(参见 Youtube )。
  19. 了解每个协议使用的标准端口(参见 PearsonCertification )用于从客户端到服务器的出站流量。短暂端口对于从服务器到客户端的响应流量的作用(参见 Techopedia )，还了解IPv4和IPv6寻址之间的差异，并了解简化从IPv4到IPv6过渡的不同技术(双栈、隧道和转换)(参见 Youtube )。
  20. 了解什么是正向代理，以及它如何用于阻止请求、缓存内容、记录日志和对服务器隐藏客户端的身份，还了解什么是反向代理，以及它如何用于对客户端隐藏服务器的身份(参见 Youtube )，还了解反向代理和负载平衡器之间的区别(参见 Youtube )。
  21. 对负载平衡器(参见 Youtube )和所使用的不同类型的路由(循环法、加权RR、最少连接数、一致散列法)(参见 KempTechnologies )以及负载平衡器如何使用HTTP头X-Forwarded-For维护原始请求者IP(参见 AWS )有一个共同的理解。
  22. 了解防火墙，有状态防火墙和无状态防火墙之间的区别，以及第7层防火墙(WAF)和第4层防火墙之间的区别(参见 Youtube )，以及如何使用防火墙在网络中创建dmz(参见 Barracuda )
  23. 知道如何使用标准网络实用程序来解决网络问题，如PING、Tracert、Telnet、ARP、Netstat、Nbtstat、NSLookup和IPConfig/IFConfig(参见 PluralSight )，以及常见实用程序的工作原理，如Ping(参见 Youtube )、Trace Routing tracert(参见 Youtube )、Telenet(参见 Youtube )。
  24. 了解如何使用WebSockets(参见 Youtube )或服务器发送的事件(参见 Youtube )在服务器和客户端之间创建全双工双向通信通道，以及这两种方法之间的区别(参见 Telerik
  25. 了解什么是内容交付网络(CDN)以及它们如何工作(参见 Youtube )以及它们如何加速内容交付，还要了解什么是存在点弹出(参见 StackPath )以及什么是边缘位置(参见 Edureka )。
