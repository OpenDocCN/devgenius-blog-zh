## 分解任务

![](img/d06776161c9ce1144f2b0ea1da4a3765.png)

照片由[马库斯·斯皮斯克](https://unsplash.com/@markusspiske?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 拍摄

大任务应该分成更小的任务。软件工程师经常这样做是为了赚大钱。你不会有太大压力，因为任务是逐步完成的。

这会给你成就感，会激励你完成整个任务。小的增量变化要好得多。立刻处理整个任务是很费时间的。

增量备份更容易估计，因此您可以更快地完成它们。一路上你看到了进步，所以它会推动你更快地完成。

## 处理讨厌的虫子

![](img/91517f2a528167f20aaf87d8a05c47fe.png)

[Martin Péchy](https://unsplash.com/@martinpechy?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

与团队成员协商，找到解决问题的方法。在日报中提到你在修复这个 bug 时遇到了麻烦。

解决不了就不要失去动力。如果您没有，其他开发人员会修复它。不修也不会失去力量。

讨厌的虫子会大大降低你的动力。一两天后，你质疑自己的知识。最好远离它，因为它会耗尽你的能量。

## 搬运测试人员

测试对于任何软件开发都是必要的。解决 bug 是工作的一半。测试人员需要批准解决方案，以解决任务。

确保你重视他们的意见，因为这将有助于长远发展。他们提出了各种方法，不像开发人员那样。

如果他们不喜欢你的解决方案，不要气馁。这只会给你一个更大的图景，并激励你去探索更多。
