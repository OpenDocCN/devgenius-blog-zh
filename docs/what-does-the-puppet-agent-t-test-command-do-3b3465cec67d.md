# puppet agent -t 或 test 命令有什么作用？

> 原文：<https://blog.devgenius.io/what-does-the-puppet-agent-t-test-command-do-3b3465cec67d?source=collection_archive---------5----------------------->

![](img/159b438241466175f437a73fda8896d0.png)

Puppet 是一个允许您管理软件配置的工具，它有自己的声明性语言。

其中一个概念是 puppet 代理守护进程，它可以基于 cron 调度自动运行，也可以出于测试目的交互运行。

要交互式地运行它，您可以使用`puppet agent`命令。

在这篇简短的文章中，我们将回顾当您将`-t`或`--test`标志添加到`puppet agent`时的含义。

首先，`-t`和`--test`是同一个东西，运行`puppet agent`的时候可以随意使用其中任何一个。所以不是 T7 就是 T8。

这个标志的意思是，傀儡代理将在前台运行一次，从傀儡主服务器获取最新的目录，然后立即在本地应用它。还会发生详细日志记录。

如果你得到一个`0`退出码，那么这个目录就成功应用了。如果您得到一个`1`错误代码，那么由于失败或缺乏尝试，目录不适用(例如，这可能来自一个已经在进行的运行)。

您之所以要运行此命令，是因为您想要手动覆盖目录，而不是等待目录的配置时间自动应用。

这里有一点很重要，至少在我看来，`-t`和`--test`并不是很好的名字。原因是它们给人的印象是使用它们会导致测试运行，而实际上它们会对运行它们的机器做出真正的改变。因此，如果您从本文中有所收获，当您实际上想要应用最新目录中的更改时，要知道使用`-t`和`--test`。
