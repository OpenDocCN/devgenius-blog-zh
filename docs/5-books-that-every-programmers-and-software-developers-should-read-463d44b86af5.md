## 如果你是一个开发人员，想提高自己的开发技能，想变得更好，这里有几本书是我推荐的。

作为一名软件开发人员，这项工作从未停止过。每天都有编写代码和解决问题的挑战。此外，还有通过阅读保持知识、紧跟潮流以及随着年龄增长保持技能不断提高的挑战。对于软件开发人员来说，风险很高，这就是为什么这五本必读的书会帮助你正面迎接这些挑战。

# 书№1

如果你是一名开发人员，你想到的第一本书是罗伯特·塞西尔·马丁的[**Clean Code。因此，这本书本质上就像是《如果你正在起步》,至少是必读的一本书。所以这本书讲的是写一个足够干净，足够有表现力的代码，我们不需要在整个代码中写一些注释，用注释来写代码。所以这一切都是为了确保你的代码写得像一首诗。所以我的意思是，你可能只写一次代码，但是其他开发人员可能会接手并检查你的代码。所以如果它是以一种有意义的方式写的，例如，当命名一个变量时，它就足够有意义，让你明白上下文是什么，这段代码或代码块应该做什么。**](https://www.amazon.com/Clean-Code-Handbook-Software-Craftsmanship/dp/0132350882)

因此，它讨论了命名变量，在编码时正确命名方法，如何构建代码，以及处理代码的更好方法。所以一定要通读这本书。

# 书№2

这本书叫做 [**神话人月**](https://www.amazon.com/Mythical-Man-Month-Software-Engineering-Anniversary/dp/0201835959) ，如果你想了解软件开发、估算、项目管理和正确的软件开发，以及软件开发中可能出错的事情，这是一本必读的书。本质上，如果你渴望成为一名项目经理，这是一本必读的书。如果你是一名开发人员或程序员，但如果你读了这本书，你会很好地理解项目管理是什么，以及从项目管理到评估和软件开发的软件工作方式。所以确保你至少读一遍这本书。

# 第三本书

下一本书是马丁·福勒 的 [**重构。所以这本书是关于让你的代码更好的。因此，如果你已经编写了代码，这本书可以帮助你不断地改进代码，以便从长远来看易于维护和调试。这本书也谈到了测试驱动开发。所以这本书给了你一个测试驱动开发的高级概述，在这里你首先写测试，然后沿着这个方向重写代码。这确保了每一行代码都是可测试的。**](https://www.amazon.com/Refactoring-Improving-Design-Existing-Code/dp/0201485672)

# 书№4

下一本书叫做 [**干净的编码者**](https://www.amazon.com/Clean-Coder-Conduct-Professional-Programmers/dp/0137081073#:~:text=In%20The%20Clean%20Coder%3A%20A,coding%20to%20refactoring%20and%20testing.) ，作者是写干净代码的罗伯特·塞西尔·马丁。这本书是关于你如何成为更好的开发人员，什么时候说是，什么时候说不是，等等。所以一定要看看，这也是一本好书。

# 书№5

你必须看的下一本书叫做 [**设计模式**](https://www.amazon.com/Head-First-Design-Patterns-Brain-Friendly/dp/0596007124) 。设计模式支持代码可读性，并允许开发人员维护和重用复杂的组件，从而减少代码大小和空间。

**维基百科对设计模式的描述是，**

> 软件设计模式是对软件设计中给定上下文中常见问题的通用可重用解决方案。

这意味着，如果你在解决一个问题时陷入困境，那么这个问题已经有了解决方案，所以你可以抓住它。所以设计模式不会先进入代码，它们会进入你的大脑，因为你需要知道在代码的什么地方插入特定的模式。所以有结构的、行为的和相关的设计模式。所以它是以一种你的大脑能够理解、挑选和掌握书中概述的概念的方式来写的，所以一定要检查一下。
