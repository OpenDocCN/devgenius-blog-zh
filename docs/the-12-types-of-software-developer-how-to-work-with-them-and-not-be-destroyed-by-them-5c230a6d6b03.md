# 1.摇滚之星(X10 开发者)

神话中的摇滚明星开发人员被安排使用新技术进行大型项目，这让其他开发人员感到紧张。摇滚明星来了，带领团队进行技术指导，完成项目。

他们需要被告知自己有多棒，感觉自己像个摇滚明星，他们需要感觉自己是这个房间里最重要的开发者。它通常以摇滚明星把他们的玩具从婴儿车扔出去，去别的地方而告终，因为他们觉得自己没有得到足够的重视。

*   优点——当受到激励时，你会表现出色，处理最大的问题并创造解决方案
*   缺点——与非摇滚明星相处不总是很好，最终会感到无聊并拂袖而去
*   就像足球罗纳尔多，他们是超级巨星，他们知道这一点。

## 如何与他们合作

让他们更多地关注技术，而不是人，因为他们会对不如他们的人或质疑他们的人感到沮丧。

向他们指出大的技术问题，并享受表演

# 2.牛仔开发者

牛仔是不喜欢开发的开发人员，对他们来说这只是一份工作。他们尽可能投入最少的努力，经常走捷径。结果是低质量的代码和以后的问题。

他们不理解或试图理解单元测试、文档、ALM、DevOps、最佳实践或标准的好处。

他们制造了很多错误，因为他们知道软件开发是关于创造质量的— [软件开发是失败者的游戏](/Software development is a losers game)

他们对发展不感兴趣，也很少改进。

*   优点——有时候有一个牛仔开发人员比没有开发人员要好，但是很少。
*   缺点——他们会让你的优秀开发人员和不在乎的人一起工作
*   就像——迈克尔·斯科特，但是没有幽默和乐趣

## 如何与他们合作

小心翼翼地，你经常会遇到这样的情况，你必须给他们诚实的反馈，他们永远不会擅长开发。他们只有足够的热情和渴望，表现一般，通常不擅长开发。

# 3.自学开发者

自学成才的开发人员通过实践自学开发。他们通常来自不同的背景，如支持，并被转移到开发。

不知道软件开发最佳实践、设计模式、坚实的原则或其他开发概念，如单元测试，除非他们不得不使用它。

他们渴望学习，但专注于编写代码，并将其他任务视为浪费时间。

优点——他们喜欢编程，渴望学习

缺点——他们通过犯错来吸取教训

就像——通过实践来学习的人

## 如何与他们合作

他们通过做、听和与其他开发人员一起工作来学习。如果你把他们和一个更有经验的开发人员联系起来，他们会进步很快。

不太可能创新，但却是他们经验的总和。

# 4.编码器

除了写代码之外的任何事情都是浪费他们的时间，他们会试图跳过其他任务或者快速完成。

这留下了空白，因为他们不理解需求、计划、编写单元测试、更新部署或编写文档。

*   优点——喜欢写代码和与代码打交道
*   缺点——开发人员只花 50%的时间写代码——[开发人员花多长时间写代码？](/how-long-do-developers-spend-writing-code-dc64b2a11511)
*   就像——加里·莱因克尔——伟大的英格兰进球者，他只是在禁区里活过来把球踢进网窝。没做别的。

## 如何与他们合作

你需要设置一个 done 的定义，这样他们就不能摆脱非编码任务。你必须清楚交付一项功能需要什么，而不仅仅是代码。

# **5。剪切和粘贴开发人员**

复制和粘贴开发人员有脆弱的知识，他们让解决方案工作，但任何改变解决方案，它打破。

复制和粘贴开发人员是快速和有效的，直到他们没有。当他们遇到麻烦时，它会受到重创，因为获得解决方案的捷径绕过了任何学习。他们没有进步，因为他们从来没有真正学习，只是在互联网上背着其他开发者。

优点——他们利用互联网的力量快速完成工作

缺点——慢慢学习，当他们陷入困境时，他们真的会陷入困境

比如——一个学生抄了朋友的作业，但被问到时却不知道写的是什么

## **如何与他们共事**

试着鼓励他们学习，祈祷一切顺利。许多开发人员通过复制和改变他人的优秀作品取得了成功。一切都很好，直到出了问题。

# 6.flash——速度与激情的开发者

闪电侠是一个开发者，他只有一个速度，100 英里每小时。他们做什么都很快；代价就是错误和 bug。flash 不会放慢速度来检查需求，仔细检查他们的工作，也很少有时间进行测试。

flash 的进展看起来很快，但是总是会有错误。使用 flash 可能会让团队成员感到沮丧。你一直要求他们慢下来，但他们走得太快，以至于不能停下来听。

*   优点——他们工作迅速，充满热情
*   缺点——随着速度而来的是错误。预计使用 Flash 会出现大量错误
*   就像——他眼中闪烁的阳光

## 如何与他们合作

让他们遵循最佳实践并完成所有步骤。代码审查对于在任何快速的错误拖慢团队的进度之前发现它们是很重要的。

# **7。《一天 24 小时的开发者》(从不睡觉)**

他们的生活正在发展，似乎一天 24 小时都在工作。白天他们努力工作，然后在你第二天工作的时候，他们已经解决了一个问题，安装了一个新工具，或者完全重写了整个代码库。

他们完成他们的工作，然后开始其他人没有想到的。有时它能感觉到有很多人。

*   优点——他们做 2 到 3 个人的工作
*   缺点——这会让团队的其他成员看起来很慢
*   就像——他们就像和埃隆·马斯克一起工作，比我们普通开发人员多工作 2 到 3 倍

## **如何与他们合作**

试着跟上并监视他们在做什么。有时他们需要专注于优先任务。

# 8.初级开发人员

我们都曾经是初级开发人员，帮助别人学习软件开发的技巧感觉很棒。当初级开发人员不断做蠢事、犯愚蠢的错误并不断提出垃圾新想法时，这种温暖模糊的感觉很快就会消失。

质疑我们是如何做事情的，问为什么是健康的，但是当你在那天回答第 50 个问题时，这就变得很烦人了。

*   优点——他们渴望学习
*   缺点——经验的代价是犯错
*   就像《第一部星球大战》中的卢克·天行者，在他成为绝地武士之前

## 如何与他们合作

这既有趣又令人沮丧。花时间指导初级开发人员，你的努力将会得到回报，因为他们会不断进步和提高。

您需要充当一个安全网，因为他们可能会在没有意识到的情况下走入软件开发的常见陷阱。如果你不快点帮忙，他们会继续挖，让事情变得更糟。

优秀的初级人才可以在支持和稳固的团队下快速进步。不要把他们放在小团队里，也不要给他们太多无人监督的责任。

# **9。做足够的显影剂**

他们来上班，做他们的工作，一天结束就离开，即使他们正在开会。

这些开发人员没有额外工作一分钟。没有闹钟显示，但他们使用一个永远不会出错的内部时钟。

试图在工作之外联系他们是没有意义的，他们永远不会回答。开发人员通常有一个不寻常的爱好，占据了周末的大部分时间，比如射箭、山地自行车或赛鸽。

*   优点——优秀的开发人员，稳定的工作
*   缺点——有时你需要完成一些事情，有时你必须工作到很晚，但不是和这个开发人员
*   就像——每天都吃同样午餐的人

## **如何与他们合作**

了解他们的局限性，给他们准确估计的工作，不需要任何额外的工作或创新的想法。

# **10。专业开发人员**

专业开发人员是伟大的多面手，有丰富的经验，并提供想法和解决方案。

专业人员没有任何明显的弱点，你可以让他们做任何开发任务，他们会有条不紊地完成任务。

始终如一，承担责任并按时完成工作。如果他们缺乏知识，他们会要求其他开发人员为他们指出正确的方向。

团队的一个资产描述了这个开发人员

*   优点——专业可靠，能完成工作，与每个人相处融洽，值得信赖
*   缺点——没什么大不了的
*   就像——如果专业人士是音乐家，他们会很冷静，没有太花哨的东西，完成工作，受到大多数人的喜欢。

## **如何与他们共事**

优秀的团队成员，对自己的工作负责，并帮助团队实现目标。

# 11。恐龙(即将退休的开发者)

恐龙开发者已经停止学习最近的开发([对开发者最大的危险](/the-greatest-danger-to-developers-82565fefb83d))并且慢慢发现开发工作更加困难。他们在缺乏知识和热情中挣扎；他们之前都看过了，不会像其他人一样在意(也许他们是对的)。

他们提供了冷静的头脑，并阻止团队恐慌和犯大错误。对这个项目不太感兴趣，因为他们已经看过很多次了。

*   优点——他们做他们的工作，通常有一些有趣的故事要讲
*   缺点——不要期望任何额外的努力，可能倾向于不认真对待工作的无聊
*   就像——你的祖父，他厌倦了生活，但却在艰难度日

# **12。缓慢而稳定的开发者**

缓慢而稳定的开发者以同样缓慢而稳定的步伐前进，不会被任何人催促。他们有自己内在的素质水平，是习惯性的动物。他们不是动态的或快速的，但他们稳定并完成工作。

如果代码需要创新，缓慢而稳定的开发人员会挣扎，他们不是领导者，但他们是好的团队成员。

*   优点——他们犯的错误更少，因为他们走得慢而且小心
*   缺点——有时候，当他们不需要坚持他们的过程时，这是令人沮丧的
*   就像加里·内维尔，一个乏味的足球运动员，与他的曼联队友相比，他很普通，但是他做好了自己的工作，并且总是在球队中占有一席之地。

## **如何与他们合作**

和他们一起工作很棒，因为他们很少有高潮，几乎没有低谷。非常适合于企业项目，在这种项目中，您希望团队保持稳定的进展，很少出现戏剧性事件。

# 结论

一个多元化的团队可以解决不同的问题，其优势在于个人的不同观点、想法和经验。

一个伟大的团队不仅仅是各个部分的总和，软件是创造性的工作，需要不同的想法来解决问题。
