数据仓库中的数据目录

原文：https://blog.devgenius.io/data-catalog-in-data-warehouse-97dc003de645?source=collection_archive---------2-----------------------#2022-11-19

如何了解数据，我们在数据仓库里有！！

在本文中，我们将讨论另一个重要的组件，数据目录。什么是数据目录，它在数据仓库中的重要性是什么，好处是什么，如何创建一个数据目录，以及可以利用的工具和技术。

什么是数据目录？

数据目录是元数据的集合，结合了数据管理和搜索工具，帮助分析师和其他数据用户找到他们需要的数据。它是所有可用数据的清单，并提供信息来评估数据对预期用途的适用性。

一个组织良好的数据仓库的数据目录为您提供快速访问信息，因为您的数据是分类的，很容易找到。您可以轻松搜索您正在寻找的数据集，查看所有可访问的数据集，并快速自信地评估和分析它们

数据目录的重要性是什么？

数据目录使您能够了解存储在数据源中的数据的技术和业务视图。数据集的自动发现不仅是不断发现新数据集所必需的，也是初始目录构建所必需的。以下几点说明了数据目录在数据仓库中的重要性:

  1. 它有助于了解我们拥有的数据类型及其用途。
  2. 谁是数据的所有者，谁负责将数据提供给仓库。
  3. 要了解哪个数据集，需要更多的安全性和法规遵从性。
  4. 了解我们是否使用了所需数据的最新版本。
  5. 有助于发现是否有不同的部门对相同的数据做类似的工作。

数据目录应该具有哪些特征:

现在我们知道了什么是数据目录及其重要性，我们将列出数据目录应该具有的重要特性。这些将帮助我们了解在构建 it 时需要注意的事项，以获得最大的 it 收益并对业务产生更大的影响。

  1. 它应该代表船上所有企业数据的唯一真实来源。
  2. 发现新数据源的能力，并确保它们如何为业务增加价值。
  3. 数据监控并帮助创建数据谱系。
  4. 实施控制以提供基于角色的访问和安全性。
  5. 如果数据质量问题出现在任何数据集中，应予以强调。
  6. 应该有能力与开源和基于云的工具和技术相结合。
  7. 易于获取数据，以便在需要时进行分析和传输。

拥有数据目录的好处:

如果我们能够构建具有上述大部分功能的数据目录，那么它将有助于获得最大收益。以下是一个好的数据目录的一些好处:

  1. 它通过提供我们拥有的所有数据的完整上下文来帮助提高数据效率。
  2. 更快的数据搜索和准备。
  3. 通过流程加速节省时间和资金。
  4. 建立对数据仓库中可用数据的信任。
  5. 帮助企业做出透明和全面的决策。
  6. 改善技术和业务团队之间的协作。

构建数据目录的工具和技术:

让我们讨论如何构建数据目录，我们可以使用哪些工具和技术。我们需要根据谁是最终用户、部署策略应该是什么、它是否支持当前和提议的工作流等来选择其中的任何一个。

Aginity (coginiti): 由于其易于理解的产品生态系统和对按需服务可扩展性选项的 SQL 友好性，它被认为是最好的数据目录软件之一。它不仅对所有公司数据进行编目，还对用于创建分析的数学进行编目。

Apache Atlas: 它为组织提供开放的元数据管理和治理功能，以构建其数据资产的目录，对这些资产进行分类和治理，并为数据科学家、分析师和数据治理团队提供围绕这些数据资产的协作功能。

data.world: 这是一个云原生平台，将直观的用户体验与强大的知识图表相结合，以提供丰富的数据发现、治理和可操作的见解。

LinkedIn data hub:data hub 是一个面向现代数据堆栈的开源元数据管理平台，支持数据发现、数据可观察性和联合治理。

Alation: Alation 的企业数据目录显著提高了分析师的工作效率，提高了分析的准确性，并推动了自信的数据驱动型决策，同时使您组织中的每个人都能够查找、理解和管理数据。

Collibra: Collibra 有助于统一个人、团队、个人、组织和系统的数据，是同类产品中最好的数据编目工具之一。它提供了具有嵌入式治理和隐私选项的高质量数据目录。

AWS 粘合数据目录:AWS 粘合数据目录是您的持久性技术元数据存储。它是一个托管服务，可以用来在 AWS 云中存储、注释和共享元数据。

Azure Data Catalog:Azure Data Catalog 是一个完全托管的云服务，让用户发现他们需要的数据源，并了解他们找到的数据源。

Google Data plex Catalog:Data Catalog 是 Dataplex 中一个完全托管的、可扩展的元数据管理服务。
