# 关键亮点

1.  **促进自主性-** 我在整个谈话中注意到的一个最突出的主题是强调在你的下属中促进自主性(在这种情况下这是一个正确的词)。作为一名经理，当你的团队在你不在场的情况下运行时，你最大的成就是。
2.  **定义流程，委托实现-** 最终，您将达到一个您的影响在日常生活中看不到的水平。你做出的决定会在更长的时间内显现出来。因此，您不想浪费时间和精力告诉您的报告每天该做什么。设定议程和标准，让你的下属(我喜欢这个词)处理如何完成这个议程。
3.  **营造交流环境-** 作为一名经理/高级开发人员/决策者，你需要营造一个交流的环境。请记住，交流不仅仅是通过演讲和每日起立鼓掌来表达的(如果你想在这些方面表现出色，请查看本文 )。代码和文档也是表达思想过程的方式。
4.  无所事事是一种策略- 无所事事有很多价值。有时候，最好的选择是坐下来，等待事情明朗起来。产生“有形”结果的压力很大，但这可能会助长一种短视的“忙碌工作”文化，即采取行动是为了填满时间，而不是实现更高的结果。
5.  **简单是商业案例的关键-** 我们经常把复杂误认为复杂。如果可以的话，你希望所有的商业案例都保持简单。**所有的利益相关者必须能够清楚地说明为什么一个特性/项目是一个好主意。**这看似微不足道，但却至关重要。

# 促进自治

当你在层级较低的时候，规则就为你设定好了。你被告知要做什么，伟大的地方会让你清楚评估标准。然而，你越往上爬，水就越浑浊。突然间，你不再得到直接的任务(“提高准确性”)，而是模糊的最终结果(“提高客户保持率”)。实现这些指令需要能够缩小视图，查看您正在处理的解决方案的各个方面。如果你是日常的微观管理，你就不能做到这一点。

只要有可能，在你的团队中创造自主精神。通过让您的报告拥有自主权，您可以立即获得许多好处-

1.  你改善了他们在你手下工作的体验。没有人喜欢微观管理者。
2.  你可以腾出时间和精力来研究更大的趋势，从而做出更好的决定。你也可以把精力集中在最需要你的地方。
3.  你给予下属成长和应对挑战的能力。他们不仅会比你做得更好(因为他们每天都在做)，而且他们会继续发展更好的技能。这将让他们茁壮成长，加强你的地位和网络。

有几种方法可以提高团队的自主性。如此之多，我会做一个单独的职位。然而，我会在这里提到两个非常强大的。记下它们，并立即实施。

首先，对于科技行业来说，我们有一条非常重要的原则——不要惩罚错误。这听起来很有趣，因为在解雇表现不佳的员工时，大型科技公司是出了名的残酷(这会导致非常高的倦怠率)。然而，如果你想灌输一种文化，让人们乐于出去承担责任，那么你就想创造一种文化，让他们觉得冒险是安全的。请记住，我们对潜在损失的恐惧远远超过对收益的热爱(大约是收益的 2.5 倍)。**如果你的员工害怕犯错，那么无论你的奖金有多丰厚，他们都会犹豫不决。**

![](img/ab72b9d681e73eb36ca84dc1eff6189c.png)

厌恶损失是我们生活中面临的最大挑战之一。我们需要克服它。[图片来源](https://www.business2community.com/brandviews/buffer/15-psychological-studies-will-boost-social-media-marketing-01070079)

相反，你想把错误当作学习的机会。一位来自亚马逊的高级人工智能经理告诉我，他的团队对错误/失误有一个简单的政策- *如果犯错的人记录了他们的错误，分析了导致错误的原因，以及如何在未来避免错误，错误就不会受到惩罚。这种方法很棒，因为它允许人们从他们(和其他人)的错误中学习。如果你开始构建内部生产力工具，那么拥有这样的日志也将帮助你设计最好的工具。*

其次，你希望你的团队非常清晰。你想让正在进行的项目的所有权非常清晰(谁负责什么等等。)您希望创建文档，使业务目标、最终客户、优先级和基本原理非常清晰。这允许你的团队成员通读并找到他们自己要攻击的区域，而不需要你手握。如果某件事非常重要，你可以在你的文档/沟通渠道中强调这一点。这样，你的团队将知道方向是什么，并采取措施适当地调整自己。这两个因素会对你的团队的自主性产生直接的影响。

然而，自主也有它的缺点。当然，它让你的团队成员有更多的自由去尝试解决方案，这提高了创新的速度，但它也产生了一个根本性的问题——它会导致混乱。没有质量检查和标准协议会导致大规模项目的失败和低效，人们可能没有时间检查每一个边缘情况。这是下一次学习的切入点。

# 定义流程，委托实施

为了平衡自主性，你还希望对你的团队施加严格的标准和质量控制，以便你的管道/用户体验的每一部分都是惊人的。在这种情况下，你能做什么？花费大量时间和精力确保每一件小事都完美无缺，或者专注于大局，冒着错过关键细节的风险，直到为时已晚？在这里，我将回顾一下我与 LinkedIn 分析主管菲德尔罗德里格斯(Fidel Rodriguez)的对话。菲德尔完美地把握住了控制和自治之间的微妙关系。

> ***强壮如竹。像竹子一样柔韧。***
> 
> *——一位同事如何描述菲德尔的管理。*

菲德尔通过为他的团队设定清晰的流程/方向，然后将日常细节留给他们来处理这种紧张关系。他对确保通过质量控制检查非常严格。但是只要他们被满足了，菲德尔就不会担心一天又一天。菲德尔特别提到了两个他要求严格的流程，你们所有人都应该执行-

1.  Gemba Walks- 这对我来说是全新的，但当 Fidel 描述它时，我觉得它棒极了。假设您的团队已经构建了一个功能。在把它运出去之前，让一个完全陌生的人问你一些关于它的问题。他们想问什么就问什么。这将有助于你像一个局外人那样看待这个特性，这对于识别改进/隐藏的缺陷是至关重要的。
2.  OKR- 当菲德尔第一次提到这一点时，我笑了一下，因为我把它与人力资源联系在一起。OKR 代表“目标和关键结果”okr 通常有一个目标，后面是一些支持性的关键结果。例如，“**我将优化这个 API，以减少 1%的错误率，减少 100 毫秒的延迟，并将每次调用的开销减少 0.5%**”[这篇文章是关于它们的很好的指南](https://www.whatmatters.com/faqs/okr-meaning-definition-example)。这给了团队更多的清晰度和方向，同时增加了整体的责任感。在努力实现这些目标的过程中，您和您的团队将会对您的解决方案的架构有更多的了解，让您更有可能取得惊人的成果。

我们已经提到了让员工拥有自主权带来的惊人好处。通过实施这些流程(以及其他质量控制检查)，您将保持产品/平台的完整性。

# 痴迷地交流

与经理们交谈时，最引人注目的一点是他们是如何强调沟通的。这种热情通过其他一些过程表现出来，他们确保他们的团队在他们开始项目工作之前就这样做了。例如，菲德尔非常重视双向交流。这是什么意思？Fidel 首先确保与项目/企业的所有利益相关者进行一对一的会谈，以充分了解他们的需求。然后，他花时间与他的报告坐在一起，非常详细地分享这些需求，听取他们的意见。这些输入可以在制定战略和设定长期目标时使用。通过这样做，整个组织团结一致，事情进展顺利。

# 什么都不做是一种选择

人们很容易想冲进项目，在事情来的时候就解决它。尤其是在科技领域，事情发展非常迅速。“快速行动，打破常规”是硅谷的一句口号。虽然速度有它的作用，但在开始/从事大型项目时，它并不是最明智的想法。

请记住，如果某个东西的规模很大，一旦你发出去，就很难修复/改变。尤其是如果你以一种非常特别的方式做事，事情一出现就被固定下来。我不是说你必须在你的预先计划中考虑每一个细节。我告诉你，盲目地投入大规模的风险投资会耗费大量的金钱和精力。

虽然花时间开会、头脑风暴和其他人力资源方面的事情很烦人，但它们之所以成为主食是有原因的。我们已经介绍了这些会议如何通过给团队一个清晰一致的方向来提高开发人员的生产力。然而，从长远来看，慢慢来，做好准备工作以获得清晰也可以节省大量资源。然而，你要确保你的薪酬结构考虑到这一点。

许多评估周期是基于一个季度内采取的行动，这可能会给想要展示结果的人带来一些压力。这导致人们做短期工作(这也更容易衡量)来获得那些奖金/而不是从公司拿提成。记住，在很多情况下，少即是多。如果你参与了大量的修复工作，你将没有精神空间和精力去创造最有效的修复。克里斯·拉夫拉在谷歌工作期间提到了提高开发人员的工作效率。如果你想知道更多，看看下面的帖子。请记住，我不久前写过那篇文章，当时我还没有写这类文章的经验。所以不如我现在写的流畅。但是，嘿，这是一个过程，如果你是我那时的高级替补之一，谢谢你在我变得更好的时候支持我。我会用最好的科技内容来回报你的信任。

# 商业案例中的简单性

很容易把复杂误认为精致和优雅。在创造你的产品时，不要犯这种错误。我已经谈到了团队中的每个人应该如何解释他们所做的事情如何融入大局。但是当谈到你的盈利策略时，我想强调这一点。

这个世界和金融体系使得在销售无价值/不可持续产品的同时实现数十亿美元的估值成为可能。经常阅读《我的金融星期五》的读者将会想起几个例子。硅谷喜欢给猪涂口红，并称之为玛丽莲·梦露。所有向你兜售耸人听闻的东西和空中楼阁的点击诱饵(主流媒体和在线专家都这么做)让情况变得更糟。不要创建参与这个系统的解决方案。

当你制定解决方案时，你应该能够非常清楚地说明这将如何帮助你赚钱/实现你的目标。你的计划越简单、越直接，这个产品就越好。货币化的效用和路径应该尽可能简单。

请注意，这并不意味着它必须很容易。无论从哪方面来说，创造下一种神奇的药物都不是一件容易的事情。然而，药物的效用是众所周知的。如果可以的话，开发具有明显和深远效用的产品。这些是在衰退中幸存下来的产品。
