让我们来认识一下编程组的新成员

V是一种全新的编程语言，在编程社区掀起了不小的波澜。

几个月前，当我阅读这篇文章时，我偶然发现了它。

虽然V在本文写作时仍处于初级阶段(在 SO 上关于它的问题只有 六个)，但它正在受到欢迎(它在GitHub上有大约 15.9K 星星)。

它也正在积极开发中(正如最近在其 GitHub repo 中提交的内容所证明的)。

所以，我想，为什么不试一试。毕竟，做一个早期采用者有什么坏处，对吗？

所以，让我们开始吧！

第一步。在本地机器上安装V

你可以用预建的二进制安装V(比如这个，也可以直接编译它的源代码。

由于我使用的是运行macOS Catalina(版本10.15.3) 的 MacBook Air (13英寸，2007年)，所以我下载了这个用于macOS的预建二进制文件。

我提取了它的内容，启动了 macOS终端 。然后， 我导航到提取预构建二进制文件的目录，并在终端中启动以下命令:

./v version

它应该显示V的当前版本，如下所示:

V 0.1.24 0d93eeb

但是，屏幕上弹出了以下提示:

你看，macOS阻止了没有经过 公证 的应用程序的执行(意思是，苹果无法验证应用程序的来源，因此，无法保证应用程序没有恶意)。

那么，我们现在做什么？放弃？

绝对不行！

有一个变通办法:我们可以告诉macOS为这种未注册的应用程序破例，方法是导航到系统首选项>安全性&隐私并点击“无论如何都允许”按钮。看一看下面的gif 图，你会有更好的想法:

既然我们已经告诉了保镖😀)让客人进来，我们应该可以开始聚会了吧？(通过终端运行V，没有任何问题)

但是没有！😯事情不是这样的。当我们再次在终端上执行./v version时，会弹出以下屏幕:

这是我们不久前遇到的同一个弹出窗口，但这一次，macOS为我们提供了“打开”按钮。让我们点击它，看看会发生什么:

只要我们单击“Open”,./v version命令就会执行并在终端上打印输出。

第二步。开始用V写代码

现在V的编译器已经在我们的本地机器上运行了，我们可以开始用V写程序了。

但是等等！难道我们不需要一个像 VSCode ，那样的花哨的IDE，以及它所有优雅的主题和插件吗😄(开个玩笑，这是我最喜欢的IDE。)

不，至少还没有！

v提供了一个【REPL】接口，让你不用在硬盘上创建任何文件就可以随意使用这种语言。

所以，按照Hello World Tradition的规定，让我们用V:

print("Bonjour V!")

按下键盘上的“回车”键，让魔法开始吧！

瞧啊。成功了！
