语境

设置者和获取者是糟糕的行业惯例。

许多 IDEs 都喜欢这种代码气味。

示例代码

错误的

class Person
{
  public string name 
  { get; set; }
}

对吧

class Person
{
 private string name 
 
 public Person(string personName)
 {
 name = personName;
 //imutable
 //no getters, no setters
 }
}
