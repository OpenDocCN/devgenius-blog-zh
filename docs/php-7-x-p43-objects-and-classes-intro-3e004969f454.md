从 PHP 5 开始，PHP 就是一种面向对象的语言。它成熟、强大，任何人都没有理由再讨厌它了。在本文中，我将讨论对象和类之间的区别，简要介绍面向对象编程的概念，并创建一个小型的介绍性类来展示类的属性、方法和局部变量。

## 类与对象

想象你正在建造一栋大楼。通常建筑师，你作为一个程序员，会开发一个蓝图，类，从这个蓝图，一个建筑将被构造，对象。当你创建一个类时，你正在开发原理图。当程序运行时，它查看这些 schematics(类)并构造对象(实例化对象)。

*   架构师=程序员
*   蓝图=类
*   建筑=物体
*   实例化=从类转换到对象

我们在程序中所做的大部分时间都是在反映真实世界的事情。由类构造的对象不一定是无生命的；我们可以模拟活的有机体或任何可以代表一个物体的想法。思考一种动物，比如狗，我们可以说狗有特征，比如:品种、身高、身长、毛色等。我们不需要仅仅模拟狗的特征；我们也可以模仿狗的动作，例如:吠叫、行走、流口水等。我们将在接下来的几篇文章中讨论狗的例子，但是现在让我们来看看类的结构。

## 阶级结构

该类将以关键字 *class* 开头，后面是类名。所有的特征(属性)和动作(方法)都在花括号之间。

属性以可见性修饰符开始， *public，private，*或 *protected* ，后面是*属性名*。我们将在后面的文章中讨论可见性修饰符。属性名可以在以后初始化或设置。从 PHP 7.4 开始，可以指定数据类型，如 string、int 等。如果你熟悉像 Java 这样的编程语言，你就已经接触过数据结构了。显式声明数据类型对于 PHP 世界来说是一个相对较新的概念。我们也将在以后的文章中讨论这个主题。

稍后将更深入讨论的另一个主题是*构造函数*。构造函数将在对象实例化期间加载。

构造函数看起来像一个函数，因为从技术上来说它就是。类内部的函数称为方法。构造函数是一种特殊类型的方法。类中的方法(动作)以可见性修饰符 public、private、protected 开始，并以与函数相同的方式声明。方法可以有可以从方法内部访问的参数。方法也可以有只能由方法本身访问的局部变量。

属性具有类范围，可以通过任何方法访问。要从方法中访问属性或其他方法，必须使用 *$this* 关键字。你只是告诉 PHP 你正在访问这个对象的*属性或方法。如果您正在访问另一个对象的属性或方法，您将实例化该对象并使用名称来访问它。在使用了 *$this* 关键字之后，使用对象操作符(- >)来检索对象的属性或方法。*

要实例化对象，必须使用 *new* 关键字。在 *new* 关键字之后，您将指定类名并在名称后面附加左括号和右括号。实例化的对象将被赋给一个变量。一旦对象被实例化，就可以用对象操作符- >访问*公共*方法和属性。
