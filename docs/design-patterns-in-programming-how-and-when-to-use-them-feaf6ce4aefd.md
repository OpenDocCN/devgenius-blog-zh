# 编程中的设计模式——如何以及何时使用它们

> 原文：<https://blog.devgenius.io/design-patterns-in-programming-how-and-when-to-use-them-feaf6ce4aefd?source=collection_archive---------1----------------------->

所以设计模式是你作为软件工程师遇到的那些语言无关的概念之一。它们基本上是对常见问题的更可重用的解决方案。对这些设计模式的良好理解会使你作为一名软件工程师为大多数问题开发更合适的解决方案变得更加容易。

在构建软件时，经常会出现一些重复的问题，并且不得不多次重写相同的代码来实现相同的目标，这不是最佳的方法，因此需要包。同样，你必须解决的许多问题都有相似的模式，这使得解决问题变得更容易，这也是设计模式产生的原因。它基本上是一堆经过测试的指导方针，总能很好地解决类似的问题。

如果你开发过任何软件，你可能会遇到一些最常见的模式，但是对每个设计模式及其用例更广泛的理解将有助于提出更聪明的解决方案。

设计模式大致分为三种:
1 .结构上，
2。行为，和
3。创造性的设计模式。

1.  结构设计模式指的是在解决一个问题时，建议通常用来组合各种类/对象的方法的模式。
2.  行为设计模式指的是帮助解决与各种类/对象之间的交互相关的问题的模式。
3.  创造性设计模式指的是基于对象所需的行为，建议通常用于从一个类创建对象的方法的模式。

这些设计模式并不严格，它们的实现也不严格，它们更像是解决类似问题的一套指导原则。没有必要总是以某种方式“束缚”自己，良好的理解只会有助于形成解决问题的方法，而不是限制它。

虽然这个介绍可能看起来很抽象，但我会从一系列文章开始，介绍一些最常见的设计模式的实现，以及如何在编程时应用它们。
