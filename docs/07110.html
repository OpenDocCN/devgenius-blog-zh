最近，我在使用一个 RecyclerView，它以一个准日历视图显示阅读计划的进度，一次显示一周。

由于每个 recycle view 项目代表一周，因此 recycle view 项目的数量(即周数)是通过将阅读计划中的天数除以 7 来计算的。在我最初的编码中，我引入了一个 bug，这个 bug 在测试中浮出了水面。

如果你没有马上发现，这是一个在 Java 和 Kotlin 中非常容易犯的整数除法错误。Java/Kotlin 在整除整数时简单地丢弃余数，本质上是执行一个floor()。所以 9 除以 7 等于 1。这意味着如果我需要显示 9 天，我的计算将只考虑第一周。上周，我实际上放弃了一半的可能情况。就令人沮丧的反复犯错误而言，对我来说，这比忘记在新应用程序中加入互联网许可标签更重要。
