帕特里克·沃德在《T21》中写道。

有时在我的本地开发工作中，有一个端口意外地被使用，我需要使用它。

例如，一个具体的用例是当我使用端口`80`进行开发时，但是当启动我的应用程序时，日志显示该端口已经被使用。

然而，当我查看是否有任何其他应用程序正在端口`80`上运行时，我找不到任何应用程序。那么是什么原因呢？

找出端口被使用的原因是另一篇文章的主题。现在，我假设你在这里的原因不是为什么，而是如何——具体来说，你如何快速释放那个端口？

# 首先，找到您想要终止的端口的 PID

首先，要在你的 Mac 上杀死某个东西，你需要找出它的 ID。

这个 ID 称为 PID。

## 什么是 PID？

PID 是进程 ID 的缩写。Mac 上的每个应用程序都运行在不同的进程 ID 上。

## find 命令是什么？

查找命令如下所示:

```
lsof -i :<portNumber>
```

因此，如果我们希望终止任何运行在`80`上的端口，我们将运行以下命令:

```
lsof -i :80
```

输出将显示`COMMAND`(实际上是应用程序的用户友好名称，如果适用的话)；`PID`(这是您需要注意的内容)；`USER`、`FD`、`TYPE`、`DEVICE`、`SIZE/OFF`、`NODE`和`NAME`。

此时只需将`PID`复制到你的剪贴板。我们将在下一个命令中用到它，也就是`kill`命令。

# 实际释放端口的 kill 命令

现在，有了剪贴板中的`PID`，你将能够`kill`它。听起来很暴力，不是吗？(如果你好奇，这里有`kill`手册页。我喜欢它有这样一段引言:“爱情不会自然死亡。它死亡是因为我们不知道如何补充它的来源。它死于盲目、错误和背叛。它死于疾病和创伤；它死于疲倦、枯萎、失去光泽”——阿奈斯·宁)

只需输入以下命令，并确保用您复制的命令替换`PID`。

```
kill -9 PID
```

`-9`只是告诉`kill`命令你正在发送一个终止信号。就是这样！
